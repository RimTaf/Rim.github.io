I will write a paragraph for each methodology of SDLC. There are many of them but I will suggest a few.
My suggestion and reasons are as follows:

  1. Waterfall Model:Waterfall model applies a step by step approach, each phase should be completed before moving to the next one. These phases usually include Requirements, Design, Development, Testing, Deployment, and Maintenance. this model is considered one of the oldest and it works best if the project is clear and with fixed requirements. the advantage of using Waterfall is its clear structure and easy documentation.
  2. Agile Model:Agile model is more flexible to work with since you can update and make changes , this is because Agile breaks the project into smaller pieces called "sprints" which makes it easier in case changes are needed.
  3. RAD (Rapid Application Development):RAD doesn't need lots of planning, starts by a basic version and after reviewing with the user changes will occurs. RAD is not suitable for big and complex projects since it requires less planning and depends on user feedback.

Among so many SDLC methodologies the ones that are in the Agile Classification are better. Among Agile SDLC Methods, I have also a few Suggestions:

  1. Scrum ​
  2. Extreme Programming (XP) ​
  3. Adaptive Software Development (ASD) ​
  4. Crystal Clear ​
